## 1. Постановка задачи

Дано: ежедневный временной ряд цены меди COMEX `P_t`, `t = 1, …, T` (где T ≈ 1300 дней ≈ 5 лет).

Требуется: для каждой даты прогноза `t = T` и каждого горизонта `H ∈ {3, 10, 21, 63, 126}` бизнес-дней оценить:

1. Точечный прогноз `P̂_{T+H}` — наиболее вероятная цена.
2. Квантильные коридоры `p_α(T+H)` для α ∈ {0.10, 0.25, 0.75, 0.90} — границы доверительных интервалов.
3. Направленческую вероятность `P(P_{T+H} > P_T)` — вероятность роста.

### Базовое преобразование: лог-доходности

Вместо абсолютных цен моделируем **лог-доходности**, что обеспечивает:
- стационарность ряда (отсутствие тренда уровня),
- свойство log-нормальности цен → удобство в построении квантилей,
- симметричность распределения относительно 0 (доходность −10% и +11.1% «равноудалены»).

Дневная лог-доходность:
```
r_t = ln(P_t / P_{t-1}) = ln P_t − ln P_{t-1}
```

Лог-доходность на горизонт H:
```
y_t^(H) = ln(P_{t+H} / P_t) = Σ_{k=1..H} r_{t+k}
```

Из прогноза `ŷ_t^(H)` точечная цена восстанавливается как:
```
P̂_{T+H} = P_T · exp(ŷ_T^(H))
```

## 3. Feature engineering

Полный набор: ~68 признаков (без COT/stocks/FRED), до ~85 с ними. Все рассчитаны **point-in-time** (для строки `t` используются только данные ≤t — нет look-ahead bias).

### 3.1. Авторегрессионные признаки

**Лаги доходностей:** для `k ∈ {1, 2, 3, 5, 10, 20, 60}`:
```
ret_lag_k_t = r_{t-k}
```

**Лаги уровня (в логах):** для `k ∈ {1, 5, 20}`:
```
log_price_lag_k_t = ln(P_{t-k})
```

### 3.2. Скользящие средние и моментум

**Простое скользящее среднее** для `w ∈ {5, 10, 20, 60}`:
```
SMA_w(t) = (1/w) Σ_{i=0..w-1} P_{t-i}
```

**Отклонение от SMA** (нормированное):
```
price_to_sma_w(t) = P_t / SMA_w(t) − 1
```

**Кроссы SMA** (стандартные технические индикаторы):
```
sma_5_20(t)   = SMA_5(t)  / SMA_20(t) − 1
sma_20_60(t)  = SMA_20(t) / SMA_60(t) − 1
```

**Моментум** для `w ∈ {5, 10, 20, 60}`:
```
mom_w(t) = ln P_t − ln P_{t-w}
```

### 3.3. Волатильность

**Историческая годовая волатильность** для окна `w`:
```
vol_w(t) = std(r_{t-w+1}, …, r_t) · √252
```

### 3.4. Технические индикаторы

**RSI(14)** — Relative Strength Index, Wilder smoothing:
```
gain_t = max(P_t − P_{t-1}, 0)
loss_t = max(P_{t-1} − P_t, 0)
EMA_gain_t = EMA_α(gain_t),  α = 1/14
EMA_loss_t = EMA_α(loss_t)
RS_t = EMA_gain_t / EMA_loss_t
RSI_t = 100 − 100/(1 + RS_t)
```

**MACD(12, 26, 9):**
```
MACD_t = EMA_12(P_t) − EMA_26(P_t)
Signal_t = EMA_9(MACD_t)
Hist_t = MACD_t − Signal_t
```

**Bollinger %B(20, 2):**
```
MA_t = SMA_20(P_t)
σ_t  = STD_20(P_t)
Upper_t = MA_t + 2σ_t,   Lower_t = MA_t − 2σ_t
%B_t = (P_t − Lower_t) / (Upper_t − Lower_t)
```

**ATR(14)** — Average True Range:
```
TR_t = max(H_t − L_t, |H_t − P_{t-1}|, |L_t − P_{t-1}|)
ATR_t = EMA_α(TR_t),  α = 1/14
atr_rel_t = ATR_t / P_t   # нормированный
```

### 3.5. Кросс-активные признаки

Для каждого актива `a ∈ {DXY, WTI, Gold, Silver, SP500, US10Y}`:
```
a_ret_1d_t   = ln(a_t / a_{t-1})
a_ret_5d_t   = ln(a_t / a_{t-5})
a_ret_20d_t  = ln(a_t / a_{t-20})
a_vol_20_t   = std(a_ret_1d, окно 20) · √252
corr_cu_a_60_t = corr(r_{t-59:t}, a_ret_1d_{t-59:t})
```

### 3.6. COT-признаки (если доступны)

Пусть `MM_t` — Money Manager net long на неделе t (forward-fill на дневной сетке).

```
cot_mm_net_long_t       = MM_t
cot_mm_net_long_z_t     = (MM_t − μ_5y(MM)) / σ_5y(MM)   # z-score за 5 лет
cot_mm_net_long_chg_4w  = MM_t − MM_{t-20}
cot_mm_net_long_chg_12w = MM_t − MM_{t-60}
cot_open_interest_t     = OI_t
cot_oi_chg_4w           = (OI_t / OI_{t-20} − 1) · 100
cot_pm_net_long_t       = PM_t   # Producer/Merchant
```

### 3.7. LME stocks (если накоплены)

```
lme_stock_total_t      = S_t
lme_stock_log_t        = ln(S_t)
lme_stock_chg_5d_t     = (S_t / S_{t-5} − 1) · 100
lme_stock_chg_20d_t    = (S_t / S_{t-20} − 1) · 100
```

### 3.8. Календарные

```
dow_t   = t.weekday()    # 0..4
dom_t   = t.day          # 1..31
month_t = t.month        # 1..12
```

### 3.9. Авто-фильтрация фич

В функции `prepare_xy` колонки с покрытием не-NaN < 50% отбрасываются автоматически. Это защита от ситуации, когда новые источники (LME stocks, FRED) ещё не накоплены — модель не «выкосит» все строки из-за одной разреженной колонки.

## 4. Модель 1: GBM (геометрическое броуновское движение)

Самая простая стохастическая модель цены. Предполагает, что лог-доходности — i.i.d. нормальные:
```
r_t ~ N(μ_d, σ_d²)
```

Оценки параметров на окне последних 60 дней:
```
μ_d = mean(r_{t-59:t})
σ_d = std(r_{t-59:t})
```

С усадкой дрейфа коэффициентом 0.5 (защита от случайных хвостовых трендов):
```
μ_d_shrunk = 0.5 · μ_d
```

На горизонте H лог-доходность распределена как:
```
y^(H) ~ N(μ_d · H, σ_d² · H)
```

Отсюда:
```
μ_T = μ_d · H
σ_T = σ_d · √H
```

### Точечный прогноз

Поскольку `P_{T+H} = P_T · exp(y^(H))` — log-нормальное распределение, его **математическое ожидание**:
```
E[P_{T+H}] = P_T · exp(μ_T + ½σ_T²)
```

(Поправка `½σ_T²` — стандартный «log-normal correction».)

### Квантили коридоров

Для уровня α (например, 0.10):
```
p_α(T+H) = P_T · exp(μ_T + σ_T · Φ⁻¹(α))
```
где `Φ⁻¹` — обратная функция стандартного нормального распределения, реализованная через Acklam-приближение (без зависимости от scipy).

### Вероятность роста

```
P(P_{T+H} > P_T) = P(y^(H) > 0) = Φ(μ_T / σ_T)
```

### Когда полезен GBM

- Когда нет фундаментальных сигналов (тёмный пол).
- Как «sanity check» — если ML-модели расходятся с GBM на 30%, скорее всего ML-модель переобучилась.
- В Turbulent режиме — GBM не выдумывает зависимости и просто отражает текущую волатильность.

## 5. Модель 2: ARIMA(1,1,1)

Классическая авторегрессия с интегрированием. Применяется на **лог-цене** (а не на доходностях напрямую), чтобы автоматически получить интегрирование первого порядка.

Модель:
```
(1 − φ_1 L)(1 − L) ln P_t = (1 + θ_1 L) ε_t,   ε_t ~ N(0, σ²)
```

где `L` — оператор сдвига, `φ_1` — AR-коэффициент, `θ_1` — MA-коэффициент.

### Оценка

Через `statsmodels.tsa.arima.model.ARIMA` на окне последних **750 дней** (≈ 3 года) методом максимального правдоподобия.

### Прогноз

`fit.get_forecast(steps=H)` возвращает:
- `predicted_mean.iloc[-1]` — точечный прогноз `ln P̂_{T+H}` на горизонт H.
- `se_mean.iloc[-1]` — стандартная ошибка прогноза.

```
ŷ_T^(H) = predicted_mean − ln P_T
μ_T = ŷ_T^(H)
σ_T = se_mean
P̂_{T+H} = exp(predicted_mean)
```

### Почему именно (1, 1, 1)

Для лог-цены меди тест ACF/PACF на ~5 годах истории показывает:
- Лаг-1 автокорреляция доходностей ~0.05 (слабая, но есть).
- Спектральная плотность близка к 1/f² — характерно для интегрированного процесса первого порядка.
- (1, 1, 1) — minimum sufficient model; (2, 1, 2) даёт сопоставимый log-likelihood, но переобучается.

### Особенность ARIMA

`predicted_mean` на длинных горизонтах быстро уходит в **константу** (среднее ряда). Это даёт ARIMA Δ ≈ 0 на 3-6 месяцев — это **корректно** и согласуется с гипотезой эффективного рынка (random walk).

## 6. Модель 3: XGBoost (Direct Multi-Step)

Стратегия «direct»: на каждый горизонт обучается **отдельный** регрессор. Цель — `y_t^(H) = ln(P_{t+H} / P_t)`. Вход — все ~68-85 фич, рассчитанных на момент `t`.

### Гиперпараметры (фиксированы для MVP)

```
n_estimators      = 400
max_depth         = 4         # неглубокие деревья → меньше переобучения
learning_rate     = 0.03
subsample         = 0.8       # стохастический градиент по строкам
colsample_bytree  = 0.8       # подмножество фич на дерево
reg_lambda        = 1.0       # L2-регуляризация
min_child_weight  = 5         # минимальный «вес» в листе
tree_method       = "hist"    # быстрая гистограммная реализация
```

### Процедура обучения

1. Из `prepare_xy(raw, H)` получаем `(X, y)` — n_obs строк, k фич.
2. Делим временно: первые 85% — train, последние 15% — val.
3. `XGBRegressor.fit(X_train, y_train, eval_set=[(X_val, y_val)])` — обучение с ранней остановкой по validation loss.
4. Predict на `X_val`, residuals `e = y_val − ŷ_val`.
5. Запоминаем `residual_std = std(e)` — основа для σ_T.

### Прогноз и калибровка коридоров

Для прогноза «сейчас» (t = T):
```
μ_T = ŷ_T^(H) = model.predict(X_T)
σ_T = max(residual_std, σ_d · √H, 1e-9)
```

**Пол σ_T через GBM** (σ_d · √H) — критически важный фикс: если validation-выборка мала, residual_std может быть занижен, что искажает квантили. Поэтому коридор не может быть **уже**, чем у GBM.

Точечный прогноз — без log-normal поправки:
```
P̂_{T+H} = P_T · exp(μ_T)
```

(GBM использует `exp(μ + 0.5σ²)`, а XGB — `exp(μ)`. Причина: residuals XGB уже учитывают условную дисперсию через train-структуру, поэтому μ_T уже «правильный» условный mean.)

### Когда XGBoost даёт прирост

- На горизонтах 1-3 месяца, когда фундаментальные сигналы (COT, stocks, корреляции) играют большую роль.
- В трендовых режимах — деревья хорошо ловят нелинейные взаимодействия фич.
- **Не** в шоковых ситуациях — при структурных break-ах XGB ломается, потому что обучен на старой связи.

## 7. Модель 4: MLP (Multi-Layer Perceptron)

Альтернативный ML-подход через нейросеть. Реализован через `sklearn.neural_network.MLPRegressor` (без зависимости от torch/TF).

### Архитектура

```
Pipeline:
  StandardScaler        # центрирование и масштабирование фич
  → MLPRegressor:
      hidden_layer_sizes = (24, 12)   # 2 скрытых слоя
      activation         = "relu"
      alpha              = 5e-2       # L2-регуляризация (сильная)
      learning_rate_init = 5e-4
      early_stopping     = True
      validation_fraction= 0.15
      n_iter_no_change   = 20
```

### Защита от выбросов

Нейронные сети без больших данных переобучаются и могут давать абсурдные прогнозы (−50% за 6 мес). Поэтому:

**Clip μ_T в пределах ±3σ_floor:**
```
σ_floor = σ_d · √H            # горизонтальная hist-vol
cap     = 3 · σ_floor
μ_T_raw = model.predict(X_T)
μ_T     = clip(μ_T_raw, −cap, +cap)
```

**Clip σ_T сверху:**
```
σ_T = min( max(residual_std, σ_floor), 2.5 · σ_floor )
```

Это даёт «sane defaults»: MLP не может предсказать движение больше 3σ горизонта (≈ 60-65% движение за 6 мес — уже почти невозможно).

### Зачем MLP при наличии XGBoost

- MLP лучше ловит **плавные** нелинейные функции (XGBoost — кусочно-постоянный).
- В ансамбле MLP и XGB дают **некоррелированные ошибки** → лучшая агрегация.
- Расхождение MLP vs XGB на одном горизонте — индикатор сложного сигнала.

## 8. Ансамбль

Взвешенная агрегация моделей в **лог-пространстве**.

### Веса по умолчанию

```
w_XGB    = 0.40   # главный ML-сигнал
w_MLP    = 0.20   # вспомогательный ML
w_ARIMA  = 0.25   # эффективный baseline
w_GBM    = 0.15   # волатильностный пол
```

В сумме 1.0. Веса настраиваются пользователем через сайдбар Streamlit.

### Агрегация μ_T и σ_T

```
μ_T_ens = Σ_i (w_i · μ_T_i) / Σ_i w_i
σ_T_ens = Σ_i (w_i · σ_T_i) / Σ_i w_i
```

### Точечный прогноз ансамбля

С log-normal поправкой:
```
P̂_{T+H}^ens = P_T · exp(μ_T_ens + ½σ_T_ens²)
```

### Квантили — те же формулы, что и для одной модели

```
p_α^ens(T+H) = P_T · exp(μ_T_ens + σ_T_ens · Φ⁻¹(α))
```

### Почему **среднее** σ_T, а не корень из суммы

Технически корректно агрегировать дисперсии:
```
σ²_ens = Σ_i w_i² σ²_i + 2 Σ_{i<j} w_i w_j ρ_{ij} σ_i σ_j
```

Но `ρ_{ij}` мы не оцениваем (ошибки моделей коррелированы, но как — неизвестно). Простое усреднение σ_T — компромисс: не раздувает коридор, как `max`, и не занижает, как `min`.

## 9. Markov-Switching Regimes

Идентификация скрытых **состояний** рынка, в которых параметры лог-доходностей различаются.

### Модель

Лог-доходность в момент `t` подчиняется одному из `K` режимов:
```
r_t | S_t = k  ~  N(μ_k, σ_k²),   k = 1, …, K
```

Скрытое состояние `S_t` — цепь Маркова первого порядка с матрицей переходов:
```
P(S_{t+1} = j | S_t = i) = p_{ij},   Σ_j p_{ij} = 1
```

«Persistence» режима = `p_{ii}` — вероятность остаться в нём.

### Оценка

`statsmodels.tsa.regime_switching.markov_regression.MarkovRegression` с параметрами:
- `k_regimes = 2` или `3`.
- `trend = "c"` — константа в каждом режиме (т.е. μ_k).
- `switching_variance = True` — разная σ_k для каждого режима.
- `order = 0` — без AR-компоненты (можно расширить).

Метод оценки — EM-алгоритм с Hamilton filter.

### Что извлекаем

1. **Параметры режимов:**
   ```
   μ_k_год = μ_k · 252 · 100   # годовая доходность, %
   σ_k_год = σ_k · √252 · 100   # годовая волатильность, %
   p_{kk}                       # persistence
   ```

2. **Сглаженные вероятности** (smoothed marginal probabilities):
   ```
   P(S_t = k | r_1, …, r_T)   ∀t, k
   ```
   Используют **всю** доступную выборку для каждой точки `t`. Сглаживают «ретроспективно».

3. **Фильтрованные вероятности** (filtered):
   ```
   P(S_t = k | r_1, …, r_t)
   ```
   Используют только данные до `t` — соответствуют real-time восприятию.

4. **Текущий режим** = argmax по последней строке smoothed_probs.

### Автолейблинг

Эвристическое именование режимов:
- Режим с минимальной σ_k → «Calm» (если μ_k > 0) или «Sideways».
- Режим с максимальной σ_k → «Turbulent» (если μ_k < 0) или «Bull-volatile».
- В k=3: средний по σ → «Trending» / «Correction».

### Применение

1. **Информационное**: показать пользователю, в каком режиме сейчас рынок.
2. **Адаптивное взвешивание** (потенциально, не реализовано): в Calm — больше веса XGB/MLP, в Turbulent — больше GBM/ARIMA.
3. **Фичи**: вероятности режимов можно подавать как фичи в ML-модели.

## 10. Walk-Forward Back-Test

Главный метод оценки достоверности прогнозов.

### Алгоритм

```
1. Параметры: train_min_days T₀, step_days Δ.
2. Для каждой точки t ∈ {T₀, T₀+Δ, T₀+2Δ, …, T − max_H}:
    a. Берём prices[1..t] как «известную» историю.
    b. Обучаем все модели на этих данных.
    c. Делаем прогноз для каждого H.
    d. Сравниваем с реальной P_{t+H} из «будущего» (которое сейчас уже известно).
3. Агрегируем метрики по всем (t, H, model).
```

Это симулирует, **что бы предсказала** модель в момент t, если бы её спросили тогда.

### Метрики

#### Ошибки прогноза

**MAE — Mean Absolute Error** (USD/lb):
```
MAE = (1/N) Σ |P̂_{t+H} − P_{t+H}|
```

**RMSE — Root Mean Squared Error**:
```
RMSE = √( (1/N) Σ (P̂_{t+H} − P_{t+H})² )
```

**MAPE — Mean Absolute Percentage Error** (%):
```
MAPE = (1/N) Σ |P̂_{t+H} − P_{t+H}| / P_{t+H} · 100
```

#### Направленческая точность

**HitRate** — доля верно угаданных направлений:
```
hit_t = 1, если sign(P̂_{t+H} − P_t) = sign(P_{t+H} − P_t), иначе 0
HitRate = (1/N) Σ hit_t · 100
```

50% = монетка. Для медь-фьючерса HitRate >55% уже неплохо (это сигнальный, не случайный шум).

#### Калибровка коридора

**Coverage80** — доля случаев, когда факт попал в коридор `[p_10, p_90]`:
```
cov_t = 1, если p_10 ≤ P_{t+H} ≤ p_90, иначе 0
Coverage80 = (1/N) Σ cov_t · 100
```

**Целевое значение = 80%**, поскольку коридор по конструкции должен покрывать 80% случаев.

- Coverage80 ≈ 80% → коридор хорошо откалиброван.
- Coverage80 > 90% → коридор слишком широкий (модель не уверена, но это не отражено в σ).
- Coverage80 < 70% → коридор слишком узкий (модель переоценивает свою точность). **Опасно** для оценки риска.

### Типичные результаты на 5-летней истории

| Горизонт | Лучший MAPE | Лучший HitRate | Лучший Coverage80 |
|---|---|---|---|
| 3 дня | ARIMA: 1.6% | Ensemble: 57% | ~93% |
| 10 дней | ARIMA: 4.3% | XGB: 53% | ~77% |
| 1 мес | ARIMA: 5.5% | Ensemble: 60% | ~80% |
| 3 мес | ARIMA: 9.0% | ARIMA: 53% | ~83% |
| 6 мес | ARIMA: 11.2% | ARIMA: 63% | ~87% |

**Вывод**: ARIMA побеждает по MAPE на всех горизонтах, что согласуется с гипотезой эффективного рынка (random walk — лучший прогноз для лога цены). Ценность ML-моделей — в направленческой точности и распределении плотности (через ансамбль).

## 11. Ограничения, допущения, риски

### 11.1. Допущения

1. **Лог-нормальность цены** на горизонте H. Это упрощение; реальные хвосты толще нормальных (kurtosis ≈ 6-10 для дневной доходности меди). Квантили p10/p90 могут **недооценивать** экстремумы.
2. **Стационарность** параметров (μ, σ) на окне 60 дней. После шоков (Cobre Panamá ноябрь 2023, тарифы Трампа июль 2025) параметры быстро меняются — модель адаптируется с лагом.
3. **Независимость моделей в ансамбле**. На самом деле ошибки скоррелированы; σ_T_ens чуть занижен.
4. **Отсутствие look-ahead**. Это проверяется в коде: фичи `t` используют только данные ≤t, target `y^(H)_t` — только `P_{t+H}`, в обучении val_set = tail.

### 11.2. Структурные риски

1. **Постпандемийный сдвиг корреляций** (Cu vs DXY: исторически −0.65…−0.82 ослабла до 0 в 2021-2022, периодически положительная). Базовые корреляционные ассумции ломаются.
2. **Разрыв COMEX vs LME из-за тарифов США 2025-2028**. HG=F (наш ряд) — американская цена; LME 3M — глобальная. Прогноз HG=F не соответствует глобальной экономике меди.
3. **Шоки предложения** (забастовки, санкции, тарифы) — невозможно предсказать из ценового ряда. Модель видит их только **после** инцидента.

### 11.3. Технические оговорки

1. **MLP клиппится ±3σ** — это аккуратный фикс, но в режиме Turbulent (реальные движения >3σ возможны) клипп будет **занижать** экстремумы.
2. **σ_floor через GBM** — в Calm режиме завышает коридор XGB/MLP (но это безопаснее, чем занижение).
3. **CFTC COT в дневной сетке через ffill** — реальная информация недельная, лаг 3 дня (release пятницы за вторник).
4. **LME stocks накапливается с нуля** через `update_data.py`. До накопления 60+ точек фича бесполезна.

### 11.4. Тесты, которые стоит делать

1. **Bai-Perron** на structural breaks в (μ_d, σ_d) — выявить точки сдвига.
2. **Granger causality** между Cu и каждой фичей (DXY, COT MM long, …) — обоснованность их в модели.
3. **ARCH-LM** на остатках — проверка наличия гетероскедастичности (GARCH-расширение для σ_T).
4. **Diebold-Mariano** для попарного сравнения моделей — статистически ли значимо различие в MSE.

## 12. Что можно улучшить (Roadmap для математики)

### 12.1. Замена нормальности на t-распределение

```
y^(H) ~ t_ν(μ_T, σ_T²)
```
ν ≈ 5-8 для меди — лучше описывает толстые хвосты. Квантили:
```
p_α = P_T · exp(μ_T + σ_T · t⁻¹_ν(α) · √((ν-2)/ν))
```

### 12.2. GARCH(1,1) вместо постоянной σ

Текущая σ_d = std последних 60 дней — без учёта volatility clustering. GARCH:
```
σ²_t = ω + α · ε²_{t-1} + β · σ²_{t-1}
```
даёт условную, **зависящую от t** волатильность. Особенно полезно для длинных горизонтов.

### 12.3. Адаптивные веса ансамбля по режиму

Веса `w_i` зависят от текущей вероятности режима:
```
w_i(t) = w_i_calm · P(calm | t) + w_i_turb · P(turb | t)
```
Калибровка `w_i_calm` и `w_i_turb` — отдельный backtest.

### 12.4. Quantile regression вместо нормальной аппроксимации

Вместо `μ + σ·Φ⁻¹(α)` — обучать отдельный XGB на каждый квантиль (objective = `quantile_alpha`). Это даст асимметричные коридоры и автоматически учтёт толстые хвосты.

### 12.5. LSTM/Transformer для коротких горизонтов

Текущий MLP — feedforward. Полноценная sequence-to-one архитектура (LSTM, TFT) могла бы лучше использовать структурную динамику ряда. Требует PyTorch/TF, более сложного обучения и регуляризации (отложено в Roadmap).

### 12.6. Включение фундаментала через структурные модели

- **Cointegration-тест Йохансена** для Cu vs USD/CLP (peso Чили), Cu vs aluminium.
- **VAR** на системе {Cu, DXY, Brent, Caixin PMI, ICSG balance} с импульсными откликами — лучшая интерпретируемость.
